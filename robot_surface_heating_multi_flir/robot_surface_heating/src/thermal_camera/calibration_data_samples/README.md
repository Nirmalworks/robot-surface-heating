# Calibration Data Samples

These are **representative samples** from the original `calibration_data/` directory,
kept here as reference so you can see what the calibration pipeline produces.

The full `calibration_data/` directory (26 MB, 596 PNGs + 42 PKLs) is excluded from
this repo via `.gitignore` — it's tied to a specific hardware setup and date and
isn't reusable.

## What's in each file

### `intrinsic/leptonN/`
Per-camera intrinsic calibration captures. Used to compute the lens/sensor model
(focal length, principal point, distortion coefficients) for a single FLIR Lepton.
Only `lepton0` images were kept in the original capture set.

- `thermal_calib_NNN.png` — A thermal frame of the chessboard calibration target
  with corner detection drawn on it. Generated by `pnp_extrinsic.py` /
  `intrinsic_calib.py` when corners were successfully detected.

### `extrinsic/<session>_<camera_index>/`
Multi-camera extrinsic calibration. Each session captures all cameras
simultaneously viewing the same target so their relative poses can be solved.
Folder names match the `extrinsics_prefix` in `config/camera_params.yaml`
(currently `03_03_26_calibration_`).

- `thermal_calib_capture_NNN.png` — Raw thermal frame as captured from the
  camera (what the operator saw when pressing the capture key).
- `thermal_calib_NNN.png` — Same frame with the detected target features
  (chessboard corners or PnP fiducial points) overlaid. Used to visually
  verify that detection succeeded before solving.
- `thermal_validate_extr_NNN.png` — A validation frame captured after the
  extrinsics were solved, with the reprojected 3D target points overlaid
  on the image. Tight overlap = good calibration.
- `extrinsics.pkl` — Final 4×4 camera-to-world transform for this camera.
  ~760 bytes. Loaded by `camera_frame_broadcaster` to publish TF.
- `calibration_data.pkl` — Intermediate solver state: the per-image
  detected 2D points, the corresponding 3D points, reprojection errors,
  etc. Useful for re-running bundle adjustment or debugging.
- `extrinsics_BA.pkl` (not always present) — Output after bundle
  adjustment refinement across all cameras jointly.

## How this fits the codebase

The calibration pipeline is implemented in:
- `thermal_camera/thermal_camera/pnp_extrinsic.py` — capture + solve
- `thermal_camera/thermal_camera/intrinsic_calib.py` — single-camera intrinsics
- `thermal_camera/thermal_camera/camera_frame_broadcaster.py` — loads
  `extrinsics.pkl` at runtime and broadcasts TF

The config in `thermal_camera/config/camera_params.yaml` controls which
calibration session is loaded (`extrinsics_prefix`) and where to find the
intrinsics/extrinsics directories.
